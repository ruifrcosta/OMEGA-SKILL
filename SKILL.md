# UNCOMPROMISING DOMAIN MANDATES

### 1. Impeccable Context & Register Setup
Before any frontend design task or file edits, OMEGA **must** establish project-specific context:
- **Context Gathering**: Run `node {{scripts_path}}/load-context.mjs` to resolve `PRODUCT.md` (required: strategic principles, tone, audience) and `DESIGN.md` (optional: design tokens). If `PRODUCT.md` is missing, empty, or placeholder (<200 chars), halt and run `teach` to populate it first.
- **Register Selection**: Classify the surface into **brand** (marketing, landing page, campaign: design IS the product) or **product** (dashboard, app shell, tool: design SERVES the product). Load the matching reference: `reference/brand.md` or `reference/product.md`.
- **Sub-Command Routing**: If the user invoked a sub-command (e.g. `craft`, `critique`, `polish`), immediately load the matching reference from `reference/<command>.md`.

### 2. The Dynamic Visual-Silence & Color Mandate
OMEGA completely bans generic "AI slop" styles (banned: Inter, Roboto, Arial, Lucide icons, sticky top navs, heavy drop-shadows, and cards).
- **Whitespace over Enclosures**: Visual organization comes strictly from whitespace rhythm (4/8dp grid), weight hierarchy, and near-invisible separators (`border-black/5`). Avoid visual card containers or borders.
- **Concentric Bezel Alignment**: All nested bezel containers ("Doppelrand") must calculate nested inner radii to match outer parameters: `rounded-[calc(OUTER-PADDING)]`.
- **Theme Physical Scene Calculation**: Dark vs. light is never a default. Before choosing, write a single sentence describing the physical scene: who uses this, where, under what ambient light, in what mood (e.g. *"SRE looking at critical server metrics on a 27-inch monitor in a pitch-black room at 3am"* forces dark mode).
- **Color Strategy**: Use HSL or OKLCH. Tint every neutral toward the brand hue (chroma 0.005–0.01). Pick an explicit color strategy before design:
  - *Restrained*: Tinted neutrals + one accent ≤10%. Product default; brand minimalism.
  - *Committed*: One saturated color carries 30–60% of the surface. Brand default for identity-driven pages.
  - *Full Palette*: 3–4 named roles used deliberately. Brand campaigns; product data viz.
  - *Drenched*: The surface IS the color. Brand heroes, campaign pages.
- **Dynamic Brand Morphing**: Colors and typography morph dynamically based on the project's target category:
  - *Healthcare*: Emerald green (`#059669`) + clinical, readable Figtree/DM Sans.
  - *Finance*: Structured Royal Blue (`#1D4ED8`) + institutional Söhne/IBM Plex.
  - *Productivity*: Violet (`#7C3AED`) + functional, clean Geist Sans.
  - *E-Commerce*: Pitch Black (`#0F0F0F`) + high-contrast serif Playfair Display.

### 3. Absolute Match-and-Refuse Bans
If you are about to write any of these, immediately halt and rewrite the element with an alternative structure:
- **Side-Stripe Borders**: Banned. Using `border-left` or `border-right` > 1px as a colored accent on cards, callouts, or alerts. Use background tints or leading icons instead.
- **Gradient Text**: Banned. Using `background-clip: text` combined with a gradient background. Use high-contrast solid typography with weight changes.
- **Glassmorphism as Default**: Banned. Blurs and glass cards used decoratively.
- **Hero-Metric Template**: Banned. The classic big number, small label, and supporting stat with a gradient accent.
- **Identical Card Grids**: Banned. Endlessly repeating cards containing an icon, heading, and text. Break monotony with asymmetric grids and varying content scales.
- **Modal as First Thought**: Banned. Modals are lazy UX. Use inline details, sliding panes, or progressive disclosures.
- **Em Dashes**: Banned in copy. Use commas, colons, semicolons, periods, or parentheses. Also ban `--`.

### 4. High-Performance Motion Mandate
Standard linear or ease-in-out transitions are completely banned.
- Enforce spring physics with high-damping cubic-beziers: `transition-all duration-700 ease-[cubic-bezier(0.32,0.72,0,1)]`.
- Entrance reveals must animate from `scale(0.95)` and `opacity: 0` to create tactile physical depth.
- GSAP ScrollTriggers must cleanly register, use scrub lag ratios for catch-up lag, use `ease: "none"` for horizontal scrolling, and completely dispose of timelines on component unmount.

### 5. Google Workspace (GWS) Mandate
- **Safe Authentication**: Enforce `GOOGLE_APPLICATION_CREDENTIALS` service JSON injection or `gws auth login`. Raw API keys are strictly banned.
- **Model Armor Sanitization**: Any public-facing write/send operations (such as drafting emails or creating documents) that process raw user inputs **MUST** wrap parameters with the Model Armor sanitization flag: `--sanitize <TEMPLATE>`.
- **Zsh History Escape**: Sheet ranges inside commands must be double-quoted to prevent Zsh shell history expansion failures (e.g. `gws sheets +read --range "Sheet1!A1:D10"`).

### 6. Database & Deployment Mandate
- **Monotonic Clustered PKs**: High-write transactional MySQL tables must utilize auto-increment integers (`BIGINT UNSIGNED AUTO_INCREMENT`). Random UUID values are strictly banned as clustered primary keys.
- **Composite Indexing**: Enforce composite indexing placing equality columns first, followed by range/sorting columns (leftmost prefix rule).
- **Secure Token Injection**: The `VERCEL_TOKEN` must strictly be injected as an environment variable in the running context. Passing `--token` directly in CLI arguments is banned.
- **Staging Preview Gates**: Deployments must default to `preview` environments. Pushes to `production` are banned unless explicitly approved by the PMO and SRE agents.

### 7. Application Security & Pentest Mandate
- **Boundary Validation**: Every system edge (HTTP endpoints, database queries, file uploads, IPC messages) must validate and sanitize data prior to internal ingestion.
- **Secure Cookies & Hashing**: Auth cookies must carry `HttpOnly`, `Secure`, and `SameSite=Strict/Lax` flags. Passwords must strictly be hashed via Argon2id.
- **Electron Sandboxing**: Enforce `nodeIntegration: false`, `contextIsolation: true`, `sandbox: true`, and custom validated `contextBridge` interfaces.
- **Exploit Validation (Shannon)**: Vulnerability reports must only log exploits verified via a containerized reproducible Proof-of-Concept (PoC).

### 8. FinOps & Self-Healing Mandate
- **Rust Token Killer (RTK)**: All diagnostic, test, and build commands must carry the `rtk` prefix to compress shell console logs and prevent AI context-window token bloat.
- **Prompt Caching**: Place highly static instructions and system templates at the absolute beginning of context inputs to trigger gateway caching checkpoints.
- **Drift Sweeps**: Verify OMEGA repositories prior to pushes using the native CLI compliance audit:
  ```bash
  python scripts/omega-cli.py audit
  ```
