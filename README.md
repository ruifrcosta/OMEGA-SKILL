# Português

O **OMEGA** é um Sistema Operacional de Engenharia de Software Autônomo e Implacável de nível Big Tech (como Stripe, Vercel, Linear). Ele unifica todas as capacidades de desenvolvimento (Frontend, Backend, Mobile, DevOps, Segurança e IA) sob uma única governança Zero-Trust e memória organizacional permanente num cofre Obsidian de 33 pastas.

## 1. Instalação

Para instalar a skill OMEGA no seu ambiente local do **Claude Code** ou **Antigravity**:

1. Clone este repositório diretamente na pasta de skills do seu agente de IA:
   ```bash
   git clone https://github.com/ruifrcosta/OMEGA-SKILL.git
   ```
2. Ou simplesmente copie o ficheiro `SKILL.md` deste repositório para o diretório de skills configurado no seu sistema (ex: `~/.gemini/config/skills/omega/SKILL.md`).
3. Certifique-se de que as dependências da CLI estão instaladas (Python 3.x e Git).

## 2. Como Utilizar

A skill OMEGA é ativada automaticamente sempre que o agente deteta tarefas de arquitetura, desenvolvimento, segurança ou infraestrutura.

### A. Ativação via Prompt
Pode invocar a OMEGA explicitamente no chat:
- *"Usa a skill OMEGA para iniciar um novo projeto Next.js."*
- *"Aciona a OMEGA para fazer uma auditoria de segurança à minha API."*

### B. Utilização da OMEGA CLI
A OMEGA inclui uma ferramenta CLI em Python (`scripts/omega-cli.py`) para gerir a conformidade do projeto:

- **Inicializar o Cofre Obsidian**:
  ```bash
  python scripts/omega-cli.py init-vault --dir ./meu-projeto
  ```
- **Criar um Registro de Decisão de Arquitetura (ADR)**:
  ```bash
  python scripts/omega-cli.py create-adr --title "nova-banco-dados" --status accepted --deciders "CTO"
  ```
- **Auditar Conformidade de Segurança e Drift**:
  ```bash
  python scripts/omega-cli.py audit
  ```

## 3. Diretivas Absolutas (Não-Negociáveis)
- **Regra "Sem Cartões" (Visual Silence)**: O design deve ser ultra-minimalista, sem caixas de cartões elevadas ou sombras pesadas. A separação é feita por espaçamento 4/8dp e tipografia.
- **Identidade Dinâmica**: Cores e fontes adaptam-se automaticamente ao domínio do projeto (ex: Esmeralda para Saúde, Azul para Finanças, Violeta para Produtividade).
- **Zero Trust**: Validação de todas as fronteiras de dados, cookies HttpOnly/Secure e hashes de senha Argon2id.

# English

**OMEGA** is an uncompromising, Big Tech-grade Autonomous Engineering Operating System (inspired by Stripe, Vercel, and Linear). It unifies all development capabilities (Frontend, Backend, Mobile, DevOps, Security, and AI) under a single Zero-Trust governance model and maintains a permanent organizational memory inside a 33-folder Obsidian vault.

## 1. Installation

To install the OMEGA skill in your local **Claude Code** or **Antigravity** environment:

1. Clone this repository directly into your AI agent's skills folder:
   ```bash
   git clone https://github.com/ruifrcosta/OMEGA-SKILL.git
   ```
2. Alternatively, copy the `SKILL.md` file from this repository to your system's configured skills directory (e.g., `~/.gemini/config/skills/omega/SKILL.md`).
3. Ensure the CLI prerequisites are installed (Python 3.x and Git).

## 2. How to Use

The OMEGA skill is automatically activated whenever the agent detects tasks involving architecture, development, security, or infrastructure.

### A. Prompt Activation
You can invoke OMEGA explicitly in the chat:
- *"Use the OMEGA skill to bootstrap a new Next.js project."*
- *"Trigger OMEGA to perform a security audit on my API endpoints."*

### B. Using OMEGA CLI
OMEGA includes a Python CLI utility (`scripts/omega-cli.py`) to manage local developer compliance:

- **Bootstrap the Obsidian Vault**:
  ```bash
  python scripts/omega-cli.py init-vault --dir ./my-project
  ```
- **Generate an Architectural Decision Record (ADR)**:
  ```bash
  python scripts/omega-cli.py create-adr --title "new-database" --status accepted --deciders "CTO"
  ```
- **Audit Security Compliance & Architecture Drift**:
  ```bash
  python scripts/omega-cli.py audit
  ```

## 3. Absolute Mandates (Non-Negotiable)
- **"No Cards" Rule (Visual Silence)**: The UI must be ultra-minimalist. Avoid raised boxes, physical containers, and heavy drop shadows. Rely strictly on whitespace (4/8dp grid) and typography.
- **Dynamic Identity**: Brand colors and typography pairings dynamically morph based on the project's target category (e.g., Emerald for Health, Royal Blue for Finance, Violet for Productivity).
- **Zero Trust**: Boundary data validation on all entrypoints, HttpOnly/Secure cookies, and Argon2id password hashing.
