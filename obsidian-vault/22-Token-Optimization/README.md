## Core Optimization Rules

To maximize AI developer output and keep API costs extremely low, all OMEGA systems and runtime agents must enforce strict context engineering practices:

### 1. The RTK (Rust Token Killer) Directive
Developers and pipeline runners **MUST** prefix all diagnostic or build shell commands with `rtk` (e.g. `rtk vitest run`, `rtk next build`). This filters out redundant telemetry output, saving up to 99.5% of input tokens on test failures and up to 87% on typescript builds.

### 2. Prompt Caching Triggers
System instructions and base templates are highly static. OMEGA prompt structures are formatted to place these static declarations at the absolute beginning of the context windows, allowing API gateways (e.g., Anthropic, Vertex) to hit cached checkpoints and bypass repetitive token computations.
