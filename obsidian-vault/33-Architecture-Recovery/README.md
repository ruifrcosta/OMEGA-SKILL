# 33 - Architecture Recovery & Drift Detection

**Domain Owner:** OMEGA Architecture & SRE Agents
**Status:** ACTIVE

This directory governs all procedures for detecting and recovering from system architecture drift, API contract divergence, security policy breaches, and undocumented database schema changes. It unifies and deprecates the isolated legacy rules of `self-healing` and `scalability`.

## Core Drift Management Rules

To prevent codebases from deteriorating under fast-paced feature cycles, OMEGA runs continuous static and runtime checks:

### 1. Pre-push Compliance Check
All OMEGA git repositories are configured with pre-push hooks that execute our automated audit sweeps:
```bash
python scripts/omega-cli.py audit
```
This scan actively flags:
- Hardcoded credentials or API keys.
- Too-permissive CORS configurations (e.g., `Access-Control-Allow-Origin: *`).
- Structural deviations from standard directories or missing index references.

### 2. SRE Outage Automated Self-Healing
Under operational service failures (e.g. Docker container crashes, deadlocks):
- Consult unified runbook procedures inside `obsidian-vault/15-Runbooks/`.
- Automated recovery systems are triggered to execute safe, sequenced Node restarts, DNS hot-swaps, or connection pool refreshes without requiring manual SRE intervention.
