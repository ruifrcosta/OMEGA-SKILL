# 24 - Agent Orchestration

**Domain Owner:** OMEGA Architecture Office
**Status:** ACTIVE

This directory serves as the centralized governance layer for all autonomous AI subagents operating within the OMEGA ecosystem. It strictly replaces all fragmented and isolated legacy systems (e.g., `.agents`, `agent-development`, `coding-agent`).

## Core Governance Rules

As defined in `ADR-0002-autonomous-agent-standards`, OMEGA dictates the following rules for AI agents:

### 1. Zero Fragmentation
- OMEGA is the sole orchestration engine.
- AI workflows cannot spawn parallel orchestrators.
- All intelligence must persist to the `obsidian-vault`.

### 2. The OMEGA Agent Prompt Standard
When defining an agent, the following Markdown/YAML structure is **MANDATORY**:

```markdown
---
name: [agent-id-lowercase-hyphens]
description: Use this agent when [explicit triggers]. Examples:
<example>
Context: [Scenario]
user: "[User Request]"
assistant: "[How OMEGA should invoke]"
<commentary>
[Why it triggers]
</commentary>
</example>
model: inherit
color: [blue|red|green|yellow|magenta|cyan]
tools: ["List", "Of", "Tools"]
---

You are the OMEGA [Agent Role].

**Your Core Responsibilities:**
1. [Core Duty 1]
2. [Core Duty 2]

**Analysis Process:**
1. [Step 1]
2. [Step 2]

**Output Format:**
- Strict JSON / Markdown / CLI standard.

**Edge Cases:**
- If X fails, execute Y.
```

### 3. Privilege & Tools (Zero Trust)
- Agents default to `inherit` for the model to optimize tokens.
- Agents must be explicitly granted tools following the principle of least privilege.
- "God-mode" root access is reserved exclusively for the OMEGA core orchestrator.

### 4. Mandatory Executive Subagents
The OMEGA runtime actively maintains definitions for the following executive subagents (stored in this directory or mapped internally):
- **CTO Agent**: Cross-domain architecture and strategic alignment.
- **Security Agent**: Penetration testing, ISO compliance, zero-trust enforcement.
- **Platform Engineering Agent**: Monorepo health, CI/CD, Turborepo caching.
- **Frontend / Backend / Mobile Agents**: Specialized domain execution.
- **Token/Cost Optimization Agent**: Constant monitoring of AI calls, cold-starts, and context windows.
