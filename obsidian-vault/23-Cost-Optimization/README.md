## Core FinOps Rules

All infrastructure provisions under OMEGA must pass through these cost-optimization gates:

### 1. Database Connection Management
Ensure database connections utilize pooling engines (PgBouncer for Postgres) or integrated proxy managers. This prevents database memory exhaustion and eliminates paid replica scale-ups due to idle connection growth.

### 2. Serverless Optimization
Compute resources allocated via serverless functions (e.g. Vercel Functions, AWS Lambda) must:
- Optimize memory allocations using measured profile metrics (over-allocating memory raises compute billing significantly).
- Implement warm-up schedulers to keep hot instances active, minimizing expensive serverless cold-start spikes.
