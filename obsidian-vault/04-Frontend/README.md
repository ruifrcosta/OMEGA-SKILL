# 04 - Frontend & Visual Engineering

**Domain Owner:** OMEGA Frontend & UX Agent
**Status:** ACTIVE

This directory governs all user interface layout, typography styling, and visual rendering standards for OMEGA applications. It physically unifies and replaces the isolated legacy guidelines from `frontend-design`, `high-end-visual-design`, and all fragmented `gsap-*` repositories.

## Core Architectural Standards

All user interfaces compiled under OMEGA must strictly comply with our high-end aesthetic mandates to completely eliminate "AI slop" patterns.

### Enforced Frameworks
- **Core UI Structure**: Next.js (App Router), React, TypeScript.
- **Styling Architecture**: Tailwind CSS with custom theme variables.
- **Global State Management**: Zustand.
- **Motion Engine**: GSAP (for complex scroll interactions) and Framer Motion (for declarative React transitions).

### Core Standards Documents
- **[[motion-choreography]]**: Complete specification of aesthetic archetypes, spring physics curves, responsive grid collapses, GPU execution rules, and GSAP/React unmount safety protocols.
- **[[ADR-0003-frontend-aesthetics-and-motion-standards]]**: Formal decision record detailing banned typography, physical nested enclosures (Double-Bezel), and hardware-accelerated rendering policies.
